# 센터분리 바코드 공백·트림 정규화

## 현황

- [`parse-center-separation.ts`](src/lib/excel/parsers/parse-center-separation.ts): `String(value).trim()` 만 적용 → 셀 **중간 공백**(`8801 2345`)은 그대로 남음
- [`upsert-center-separation-barcodes.ts`](src/services/center-separation/upsert-center-separation-barcodes.ts) 등 서비스: 각각 `.trim()` 중복
- 입고 템플릿 바코드는 이미 [`filter-inbound-template.ts`](src/lib/excel/generators/filter-inbound-template.ts)에서 `trim().replace(/\s/g, "")` 사용

## 정규화 규칙 (통일)

신규 [`src/lib/center-separation/normalize-barcode.ts`](src/lib/center-separation/normalize-barcode.ts):

```ts
export function normalizeCenterSeparationBarcode(value: unknown): string {
  if (value === null || value === undefined) return "";
  if (typeof value === "number" && Number.isFinite(value)) {
    return String(Math.trunc(value)).replace(/\s/g, "");
  }
  return String(value).trim().replace(/\s/g, "");
}
```

- `null` / `undefined` / 공백만 → `""` (빈 행 스킵)
- 앞뒤·중간 공백·탭 등 `\s` 전부 제거
- 엑셀 숫자 셀은 정수 문자열로 변환 (과학적 표기 방지)
