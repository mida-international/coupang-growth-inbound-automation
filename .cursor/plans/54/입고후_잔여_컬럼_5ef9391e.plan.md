# 대시보드「입고후 잔여」컬럼 추가

## 계산식

```
입고후 잔여 = max(0, 샵플링재고 − 쿠팡그로스 입고추천)
```

| 기존 필드 | SQL | 비고 |
|-----------|-----|------|
| 샵플링재고 | `shopling_available_stock` | |
| 쿠팡그로스 입고추천 | `growth_inbound_recommend` | override 반영: `COALESCE(o.growth_inbound_recommend_qty, v.calculated_growth_inbound_recommend)` |

```sql
GREATEST(
  0,
  v.shopling_available_stock
    - COALESCE(o.growth_inbound_recommend_qty, v.calculated_growth_inbound_recommend)
) AS remaining_after_inbound
```

편집 모드에서는 `displayRows`가 `growthInboundRecommend`를 draft로 덮어쓰므로, UI 렌더는 **행 필드 기준 런타임 계산**으로 처리합니다 (`Math.max(0, shoplingAvailableStock - growthInboundRecommend)`). 입고추천 수정 시 잔여가 즉시 반영됩니다.

## 데이터 레이어 (뷰 + CTE)

대시보드 목록은 뷰를 직접 조회하지 않고 [`inbound-workbench-query-sql.ts`](src/services/inbound-workbench/inbound-workbench-query-sql.ts)의 `workbench_display` CTE를 사용합니다. **두 곳 모두** 동일 식을 추가합니다.

### 1. 신규 마이그레이션

`inbound_workbench_display_v` 재생성 ([`20260621120000_inbound_trends_views/migration.sql`](prisma/migrations/20260621120000_inbound_trends_views/migration.sql) 패턴):

- `DROP VIEW IF EXISTS inbound_trends_row_v` (의존 뷰)
- `DROP VIEW IF EXISTS inbound_workbench_display_v`
- `CREATE VIEW inbound_workbench_display_v` — 기존 컬럼 + `remaining_after_inbound`
- `CREATE VIEW inbound_trends_row_v` — 기존과 동일하게 재생성

### 2. 인라인 CTE

[`buildWorkbenchDisplayCtes`](src/services/inbound-workbench/inbound-workbench-query-sql.ts)의 `workbench_display` SELECT에 동일 컬럼 추가.

### 3. 목록 매핑

[`list-inbound-workbench.ts`](src/services/inbound-workbench/list-inbound-workbench.ts):

- `RawWorkbenchRow`에 `remaining_after_inbound: number`
- SELECT / `mapRow`에 `remainingAfterInbound` 매핑

## 앱 레이어

| 파일 | 변경 |
|------|------|
| [`types.ts`](src/services/inbound-workbench/types.ts) | `remainingAfterInbound: number` |
| [`inbound-workbench-sort.ts`](src/services/inbound-workbench/inbound-workbench-sort.ts) | `remainingAfterInbound`를 sort column에 추가 (`growthInboundRecommend` 다음) |
| [`build-inbound-workbench-order-by.ts`](src/services/inbound-workbench/build-inbound-workbench-order-by.ts) | `d.remaining_after_inbound` |
| [`inbound-workbench-column-layout.ts`](src/services/inbound-workbench/inbound-workbench-column-layout.ts) | `DEFAULT_COLUMN_WIDTHS.remainingAfterInbound: 96`, `DEFAULT_COLUMN_ORDER`에 삽입 |
| [`inbound-workbench-columns.tsx`](src/components/inbound-workbench/inbound-workbench-columns.tsx) | 라벨 `입고후 잔여`, 우측 정렬, `renderCell`에서 위 계산식 사용 |

열 숨김·순서·너비는 기존 `InboundWorkbenchColumnLayout` 흐름을 그대로 따릅니다. 신규 열 ID `remainingAfterInbound`가 sort/layout 상수에 들어가면 자동 연동됩니다.

## 컬럼 배치

`growthInboundRecommend` 바로 다음, `actualPackedQty` 앞:

```
... → growthInboundRecommend → remainingAfterInbound → actualPackedQty → ...
```

## 범위 외

- 편집 불가 (읽기 전용 계산 열)
- 산출물 다운로드·추세조회 등 다른 화면에는 미포함
- `inbound_workbench_v` (base view) 변경 없음 — display 레이어에서만 계산

## 검증

1. `/` 대시보드에「입고후 잔여」열 표시
2. 값 = max(0, 샵플링재고 − 입고추천) 확인 (override 행 포함)
3. 편집 모드에서 입고추천 수정 시 잔여 즉시 갱신
4. 열 헤더 클릭 정렬 동작
5. 열 숨김 메뉴에 포함
