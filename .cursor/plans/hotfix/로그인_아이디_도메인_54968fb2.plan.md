# 로그인 아이디 + @mida.com 자동 완성

## 목표

- 로그인 시 `admin`처럼 **아이디만** 입력해도 `admin@mida.com`으로 로그인
- `user@mida.com`처럼 **이미 `@`가 포함된 값**은 변환 없이 그대로 사용 (기존 전체 이메일 입력도 유지)

## 현재 구조

[`src/components/auth/login-form.tsx`](src/components/auth/login-form.tsx) — 입력값을 그대로 Supabase에 전달:

```ts
await supabase.auth.signInWithPassword({ email, password });
```

- 라벨: `이메일`, `type="email"`, placeholder `admin@example.com`
- `type="email"`은 `@` 없는 아이디만으로는 HTML5 검증에 걸릴 수 있음

## 구현

### 1. 정규화 유틸 추가 — [`src/lib/auth/normalize-login-email.ts`](src/lib/auth/normalize-login-email.ts) (신규)

```ts
const LOGIN_EMAIL_DOMAIN = "@mida.com";

export function normalizeLoginEmail(input: string): string {
  const trimmed = input.trim();
  if (!trimmed) return trimmed;
  if (trimmed.includes("@")) return trimmed;
  return `${trimmed}${LOGIN_EMAIL_DOMAIN}`;
}
```

- 공백 trim
- `@` 포함 시 그대로 반환 (다른 도메인 전체 이메일도 허용)
- 미포함 시 `@mida.com` 접미

### 2. 로그인 폼 수정 — [`src/components/auth/login-form.tsx`](src/components/auth/login-form.tsx)

| 항목 | 변경 |
|------|------|
| import | `normalizeLoginEmail` |
| submit | `email: normalizeLoginEmail(email)` 로 `signInWithPassword` 호출 |
| 라벨 | `이메일` → `아이디` |
| input `type` | `email` → `text` (아이디만 입력 가능) |
| `autoComplete` | `email` → `username` |
| placeholder | `admin` (또는 `your-id`) |

에러 메시지(`이메일 또는 비밀번호가 올바르지 않습니다.`)는 범용 문구라 **유지**.

## 변경 범위 밖
