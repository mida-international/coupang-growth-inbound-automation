# 목록 테이블 헤더 불투명화

## 원인

6개 목록 테이블의 sticky 헤더가 `bg-muted/40`(40% 불투명)을 사용합니다. 스크롤 시 뒤쪽 행 텍스트가 헤더를 **투과**해 겹쳐 보입니다.

```tsx
// 현재 (예: inbound-workbench-table.tsx)
<TableHeader className="sticky top-0 z-20 bg-muted/40">
```

`thead`에만 배경을 주면 브라우저마다 `th` 셀 단위 sticky 페인팅이 불완전할 수 있어, **`th`에도 불투명 배경**을 함께 적용합니다.

## 해결 방향

공통 상수 1곳에서 관리하고, 6개 테이블에 일괄 적용합니다.

### 신규: [`src/components/data-list/list-table-header.ts`](src/components/data-list/list-table-header.ts)

```tsx
export const LIST_TABLE_HEADER_CLASS =
  "sticky top-0 z-20 bg-muted [&_th]:bg-muted";

export const LIST_TABLE_STICKY_HEAD_CLASS = "bg-muted";
```

- `bg-muted`: 테마 기준 **완전 불투명** 배경 (라이트/다크 모두 대응)
- `[&_th]:bg-muted`: 모든 `TableHead` 셀에도 동일 배경 적용

추세조회 sticky left 열은 별도 상수:

```tsx
export const LIST_TABLE_STICKY_LEFT_HEADER_CLASS =
  "sticky top-0 z-30 bg-muted [&_th]:bg-muted after:...";
```

### 추세조회 추가 사항

[`trends-table.tsx`](src/components/inbound-trends/trends-table.tsx)는 2단 헤더 + 좌측 sticky 열이 있어 교차점 `z-30` 유지:

- `STICKY_LEFT_HEADER_CLASS`: `bg-muted/40` → `bg-muted`
- `STICKY_HEADER_CLASS`: `bg-muted/40` → `bg-muted`
- 좌측 sticky **본문** 셀(`bg-background`)은 이미 불투명 — 변경 없음

## 범위 외

- [`seller-accounts-table.tsx`](src/components/coupang-seller-accounts/seller-accounts-table.tsx), [`members-table.tsx`](src/components/members/members-table.tsx) — 목록 스크롤 영역 미사용, 이번 작업 제외
- `table.tsx` 전역 기본값 변경 없음 — 목록 테이블만 영향

## 검증

1. `/`, `/trends` 등에서 테이블 세로 스크롤 시 헤더 뒤 행이 비치지 않는지
2. 추세조회: 좌측 sticky 열 + 상단 헤더 교차점이 깨지지 않는지
3. 다크 모드에서 헤더 배경이 데이터 행과 구분되는지
4. `npm run build` 통과
